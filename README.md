# 考搭子 KaoBuddy

考搭子是一个给考前冲刺用的手机学习工作台。

我们想做的不是又一个复杂学习系统，而是一个能在临近考试时马上派上用场的小工具：把散落的课件、笔记、PDF、手写照片和视频字幕收进来，再用你自己的 AI API 帮你拆计划、讲重点、刷题和模考。

第一版先做成 PWA。手机浏览器打开就能用，也可以添加到桌面，当成一个轻量 App。

## 为什么做它

考前最难受的地方，往往不是“完全不会”，而是东西太散、时间太少、优先级不清楚。

- 课件、复习提纲、往年题分散在不同地方。
- 手写笔记拍在相册里，复习时很难重新利用。
- B站课程看过，但没有整理成可复习的文字。
- 剩下几天才开始冲刺，不知道先补哪一块最划算。
- 想用 AI 帮忙，但又希望 API、数据和节奏都掌握在自己手里。

考搭子的目标是把这些东西串成一条简单的备考流程：**建项目 -> 导资料 -> 生成计划 -> 学重点 -> 刷题 -> 模考查漏**。

## 第一版有什么

- **考试项目**：填写科目、考试日期、每天可学习时间、目标分数。薄弱项可填可不填。
- **资料库**：支持粘贴文本、Markdown、TXT、PDF。
- **手写笔记**：上传手写图片或 PDF，图片可以通过用户自己的视觉模型/OCR API 识别成文字。
- **视频链接**：粘贴 B站等公开视频链接，优先读取标题、简介和公开字幕。
- **AI 计划**：按剩余时间、资料内容和学习时长生成冲刺路线。
- **AI 讲解**：支持零基础讲解、高频考点、例题和易错点整理。
- **刷题练习**：根据资料生成练习题，也可以让 AI 批改答案。
- **短模考**：生成手机上能完成的小模考，考后给查漏建议。
- **本地存储**：项目、资料和 AI 输出默认存在浏览器 IndexedDB。
- **导出导入**：支持 JSON 备份，方便手动迁移。

## 自己的 API

考搭子第一版按 OpenAI-compatible 的接口来接模型。只要服务商兼容 `/chat/completions`，就可以尝试接入。

内置预设：

- DeepSeek
- Kimi 国内
- Kimi 国际
- OpenAI
- 自定义 OpenAI-compatible

需要填写：

- `Base URL`
- `Model`
- `API Key`

API Key 默认只保存在当前浏览器本地。请求 AI 时，后端只负责临时转发，不写数据库，也不主动打印到日志。

## 本地运行

后端：

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -e ".[test]"
uvicorn backend.app.main:app --reload
```

前端：

```bash
npm install
npm run dev
```

打开：

```text
http://localhost:5173
```

## 测试

```bash
pytest
npm run build
```

## 免费部署思路

第一版可以用免费额度先跑起来：

- 前端：Cloudflare Pages 或 Vercel。
- 后端：Render Free、Railway 免费额度，或者 Vercel Python Functions。

如果只是自己用，本地跑后端也可以。  
如果要给别人用，需要明确告诉用户：他们填写的 API Key 会经过后端转发给模型服务商。

## 当前边界

- B站视频只做公开信息和公开字幕的 best-effort 读取，不下载视频。
- 如果视频没有公开字幕，可以手动粘贴字幕、简介或课程重点。
- 手写 PDF 如果本身有文字层，会优先提取文字；纯扫描 PDF 后续再加强逐页转图识别。
- 第一版不做账号、云同步和多人协作，先把单人备考闭环做顺。

## 发布目标

`v0.1.0` 的目标是跑通最小闭环：

1. 创建考试项目。
2. 配置自己的 API。
3. 导入文本、PDF、手写笔记和视频链接。
4. 生成计划、讲解、练习和短模考。
5. 本地保存并支持 JSON 备份。
