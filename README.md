# 考搭子 KaoBuddy

一个手机也能用的免费备考小工具。

它不是 VSCode 插件，也不要求你买什么内置服务。你打开网页，建一个考试项目，把课件、笔记、PDF、手写照片、B站视频链接这些资料丢进去，再填自己的 AI API Key，就可以让它帮你做冲刺计划、讲知识点、出题、批改和模考。

第一版先做成 PWA：手机浏览器能用，也可以添加到桌面，当成一个轻量 App 用。

## 它想解决什么

很多时候不是完全不会学，而是考前东西太散：

- 课件在一个地方，往年题在一个地方，手写笔记又拍在相册里。
- B站视频看了不少，但没有整理成能复习的文字。
- 离考试没几天，不知道先救哪一块。
- 想找 AI 帮忙，但又不想被绑在某个付费产品里。

考搭子的思路很简单：资料先集中到本地，然后用你自己的 API 变成一条能执行的备考流程。

## 现在支持的功能

- **考试项目**：填写科目、考试日期、每天能学多久、目标分数。薄弱项可填可不填，不填也能开始。
- **资料导入**：支持直接粘贴文本、Markdown、TXT、PDF。
- **手写笔记导入**：上传手写图片或 PDF，图片会调用你自己的视觉模型/OCR API 识别成文字。
- **视频链接导入**：粘贴 B站等公开视频链接，优先抓标题、简介和公开字幕；不下载视频。
- **AI 制定计划**：结合考试时间、资料、学习时长，生成冲刺路线和每日任务。
- **AI 讲解**：按零基础、高频考点、例题、易错点来讲。
- **刷题练习**：根据资料生成练习题，也可以让 AI 批改答案。
- **短模考**：生成手机上能完成的小模考，考后给查漏建议。
- **本地保存**：项目、资料和 AI 结果默认存在浏览器 IndexedDB。
- **导出导入**：支持 JSON 备份，换设备时可以手动迁移。

## 支持哪些 API

第一版按 OpenAI-compatible 的格式接入。也就是说，只要服务商兼容 `/chat/completions`，基本都能试。

内置预设：

- DeepSeek
- Kimi 国内
- Kimi 国际
- OpenAI
- 自定义 OpenAI-compatible

你需要自己填：

- `Base URL`
- `Model`
- `API Key`

API Key 默认只保存在当前浏览器本地。请求 AI 时会临时发给后端转发，不写数据库，也不会主动打印到日志里。

## 本地跑起来

后端：

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -e ".[test]"
uvicorn backend.app.main:app --reload
```

前端：

```bash
npm install
npm run dev
```

然后打开：

```text
http://localhost:5173
```

## 测试

```bash
pytest
npm run build
```

如果你本机没有全局 npm，可以先装 Node.js。这个项目本身不依赖 VSCode 插件。

## 免费部署思路

想省钱的话，第一版可以这样放：

- 前端：Cloudflare Pages 或 Vercel 免费额度。
- 后端：Render Free、Railway 免费额度，或者 Vercel Python Functions。

如果只是自己用，本地跑后端也行。  
如果要给别人用，记得明确告诉用户：他们填的 API Key 会经过你的后端转发到模型服务商。

## 关于 B站视频

视频导入是尽量抓公开信息和公开字幕。

抓不到也正常，原因可能是：

- 视频没有公开字幕。
- 页面需要登录或 Cookie。
- B站页面结构变化。
- 平台反爬限制。

第一版不会下载视频，也不做绕限制的东西。抓不到字幕时，可以手动粘贴字幕、简介或课程重点，一样能进入资料库。

## 关于手写笔记

图片会走用户自己的视觉模型/OCR API。  
PDF 如果本身有文字层，会优先提取文字；如果是纯扫描件，后面可以继续加强成逐页转图识别。

第一版先追求“能把手写资料放进备考流程里”，不做复杂笔迹编辑。
