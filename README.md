# 考搭子 KaoBuddy

考搭子是一个给考前冲刺用的手机学习工作台。

我们想做的不是又一个复杂学习系统，而是一个能在临近考试时马上派上用场的小工具：把散落的课件、笔记、PDF、手写照片和视频字幕收进来，再用你自己的 AI API 帮你拆计划、讲重点、刷题和模考。

现在它是一个本地可直接打开的 PWA。双击启动脚本后，浏览器会自动打开考搭子；手机和电脑都按同一套页面来设计，Mac 和 Windows 都有本地启动入口。

## 为什么做它

考前最难受的地方，往往不是“完全不会”，而是东西太散、时间太少、优先级不清楚。

- 课件、复习提纲、往年题分散在不同地方。
- 手写笔记拍在相册里，复习时很难重新利用。
- B站课程看过，但没有整理成可复习的文字。
- 剩下几天才开始冲刺，不知道先补哪一块最划算。
- 想用 AI 帮忙，但又希望 API、数据和节奏都掌握在自己手里。

考搭子的目标是把这些东西串成一条简单的备考流程：**建项目 -> 导资料 -> 生成计划 -> 学重点 -> 刷题 -> 模考查漏**。

## 现在有什么

- **考试项目**：填写科目、考试日期、每天大概可学习时间、目标分数。薄弱项可填可不填。
- **项目侧边栏**：创建后每个科目都会出现在左侧，点一下就能进入对应项目空间。
- **模块计划**：AI 计划可以拆成知识点模块，模块能在待学、学习中、已完成之间拖拽调整。
- **资料库**：支持批量上传 PDF、DOC、DOCX、TXT、Markdown，也能手动粘贴补充文本。
- **PDF 识别**：PDF 会读取文字层，也可以在有 API Key 时识别页面里的截图、表格、图示、公式和扫描内容。
- **手写笔记**：上传手写图片或 PDF，图片可以通过用户自己的视觉模型/OCR API 识别成文字。
- **视频链接**：粘贴 B站等公开视频链接，优先读取标题、简介和公开字幕。
- **AI 计划**：按考试信息和资料内容生成知识模块计划，确认后拆成短卡片。
- **AI 讲解**：支持零基础讲解、高频考点、例题和易错点整理。
- **刷题练习**：根据资料生成练习题，也可以让 AI 批改答案。
- **短模考**：生成手机上能完成的小模考，考后给查漏建议。
- **错题和薄弱项**：练习反馈可以沉淀成错题，薄弱项和模考记录也能手动保存。
- **本地存储**：项目、资料和 AI 输出默认存在浏览器 IndexedDB。
- **导出导入**：支持 JSON 备份，方便手动迁移。

## 自己的 API

考搭子按 OpenAI-compatible 的接口来接模型。只要服务商兼容 `/chat/completions`，就可以尝试接入。

内置预设：

- DeepSeek
- Kimi 国内
- Kimi 国际
- OpenAI
- 自定义 OpenAI-compatible

日常只需要选平台、填写 `API Key`。`Base URL` 和 `Model` 已经放进高级设置里，默认 DeepSeek 会自动使用：

- `https://api.deepseek.com`
- `deepseek-chat`

API Key 默认只保存在当前浏览器本地。请求 AI 时，后端只负责临时转发，不写数据库，也不主动打印到日志。

如果还没有 DeepSeek API Key，可以去 DeepSeek 开放平台创建一个。复制后粘到考搭子里就行，不要把自己的 Key 发到群里或截图里。

## 直接使用：Mac

```text
/Users/Zhuanz/Documents/kaoBuddy/open-kaobuddy.command
```

它会自动启动本地服务，并打开：

```text
http://127.0.0.1:8000
```

打开页面后，按这个顺序用：

1. 在 **API 设置** 里选平台、填自己的 `API Key`，点“测试连接”。
2. 创建考试项目，薄弱项可以先不填。
3. 导入资料：批量上传 PDF/DOC/DOCX/TXT/MD，粘贴补充文本，上传手写图片，或粘贴 B站视频链接。
4. 点“生成计划”，检查 AI 给出的结果，再确认拆成知识点模块。
5. 在计划页拖动模块顺序和状态，学完一个就标记完成。
6. 练习、模考后保存错题、薄弱项和模考记录。
7. 需要备份时点“导出 JSON”。

## 直接使用：Windows

在 Windows 上双击项目里的：

```text
open-kaobuddy.bat
```

它会自动创建 `.venv`、安装后端依赖、打开浏览器并启动本地服务。打开的地址同样是：

```text
http://127.0.0.1:8000
```

## 手动运行

后端：

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -e ".[test]"
uvicorn backend.app.main:app --reload
```

打开：

```text
http://127.0.0.1:8000
```

现在默认页面来自 React/Vite 构建产物，FastAPI 会直接托管 `backend/static/index.html`。

## 免费部署思路

可以用免费额度先跑起来：

- 前端：Cloudflare Pages 或 Vercel。
- 后端：Render Free、Railway 免费额度，或者 Vercel Python Functions。

如果只是自己用，本地跑后端也可以。  
如果要给别人用，需要明确告诉用户：他们填写的 API Key 会经过后端转发给模型服务商。

## 当前边界

- B站视频只做公开信息和公开字幕的 best-effort 读取，不下载视频。
- 如果视频没有公开字幕，可以手动粘贴字幕、简介或课程重点。
- 普通课件 PDF 会读取文字层，也会尝试识别页面画面；识别质量取决于你接入的模型能力和课件清晰度。
- 当前不做账号、云同步和多人协作，先把单人备考闭环做顺。

## 发布目标

`v0.2.0` 的目标是把一次性计划升级成项目化的学习闭环：

1. Mac / Windows 都能双击打开。
2. 只填平台和 API Key 就能使用默认模型。
3. AI 计划可以先确认，再拆成知识点模块。
4. 知识点模块能拖拽调整顺序和状态。
5. 项目总览能按模块完成情况展示整体进度。
6. 错题、薄弱项和模考记录能沉淀下来。
7. 本地保存并支持 v2 JSON 备份。
