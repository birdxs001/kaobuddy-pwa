# Design

## Intent

KaoBuddy 的界面方向是高级产品感，不幼稚，不后台，不炫技。整体要像一个成熟的学习工作台：清楚、亲切、有冲刺感，但不会让用户觉得被视觉噪音追着跑。

## Visual System

- 主识别色保留深绿，但从旧版偏土的绿调整为更沉稳的墨绿。
- 背景从奶油纸感改成更干净的浅中性色，带一点绿色倾向，减少复古感。
- 强调色使用克制的暖橙红，只用于当前状态、完成、危险或高优先级，不到处装饰。
- 卡片和面板使用低饱和表面色、细边界和少量有方向的阴影，不做厚重玻璃效果。
- 圆角保持产品工具感：按钮和标签可以偏圆，面板和卡片控制在 10 到 16px。

## Typography

- 用系统 UI 字体栈承载中文可读性，避免依赖外部字体加载。
- 标题更有存在感，但不使用夸张流体大字。工作台标题比首页标题更收敛。
- 数字和进度使用 tabular nums，方便扫读倒计时、分钟数和百分比。
- 说明文字行长控制在 65 到 75 个中文字符附近，避免宽屏上一行太长。

## Layout

- 首页第一屏是产品级引导页：品牌、价值、真实设置表单和流程入口在同一个清楚的桌面构图里。
- 工作台保留左侧项目栏 + 右侧工作区。左侧更像项目驾驶区，右侧更像学习任务面板。
- 总览页优先展示倒计时、完成度、当前重点、最近模考和下一步行动。
- 资料、计划、模拟考、速背页面保持现有功能入口，但统一成更成熟的组件语言。
- 桌面宽屏优先展示，移动端侧栏自然折叠为顶部区域，标签和看板可横向滚动。

## Components

- Primary button：深绿底，白字，hover 有轻微抬起和色彩加深。
- Secondary button：浅色底，深绿字，适合次级操作。
- Danger button：暖红色系，文字明确，不用刺眼高饱和红。
- Panel：低饱和表面、细边界、轻阴影，避免边框 + 大阴影的通用卡片感。
- Status toast：不再使用粗侧边条，用完整背景和图标感状态区分。
- Tabs：像产品里的分段导航，当前态清楚，但不过度抢视觉。
- Empty state：给下一步提示，例如“先去计划页让 AI 生成计划”，不要只说暂无。

## Motion

动效只服务关键状态：

- 页面和面板进入使用轻微位移 + 透明度，150 到 240ms。
- 按钮 hover / active 有细小位移，按下有压感。
- 上传、生成、完成、速背斩卡片使用状态反馈动效。
- 进度条和圆环用平滑过渡。
- `prefers-reduced-motion: reduce` 时关闭大部分移动，只保留必要状态变化。

## Logo

重做为高级字母标，围绕 `K / KaoBuddy / 考` 的抽象关系展开。不要角色、不要卡通、不要文具图案。图标要在 PWA favicon 和桌面展示里都清楚，主色沿用深绿，辅色用克制浅金或暖白。
